BlockPress Protocol v1.3May 10, 2018

Last change (1.3): Added Blog Post commands Last change (1.2): Added Create Media Post command

The Protocol uses P2PKH addresses and all actions are stored on-chain with OP_RETURN and data in payloads are UTF-8 encoded.

For example, the Set Name Action (8d01) will set your name to "nakamoto":

OP_RETURN (HEX): 6a028d01086e616b616d6f746f

Breaking it down:

6a = OP_RETURN
02 = PUSH Bytes (to make it easier for interpretors to process prefix)
8d01 = Set Name Action
08 = Length of bytes for first (and only parameter in this case)
e616b616d6f746f = "nakamoto" encoded as UTF-8 bytes

Detailed Action Descriptions

Set Name

Prefix: 0x8d01
Payload Parameters:
  • Text (77 bytes) - Profile name

Example:
OP_RETURN (HEX) 6a02 8d01 086e616b616d6f746f
Sets name to "nakamoto"

Create Post

Prefix: 0x8d02
Payload Parameters:
  • Text (77 bytes)

Example:
OP_RETURN (HEX): 6a02 8d02 0c48656c6c6f20576f726c6421
Creates a post with body "Hello World!"

Like a Post (Optionally Tip Author)

Prefix: 0x8d04
Payload Parameters:
  • Txhash (32 bytes) - Post being liked / gifted.
  • Text (45 bytes) - (Unused currently)

Important Note: Tipping is completely native by creating an Output to the author's public address in the amount of the tip . See sample below.

Example:
OP_RETURN (HEX) 6a02 8d04 20829f421e381e4b84e4f1ac4ac121767eeb41d4280c555dc4cb1ae952307ee78f
Creates a like for a post/reply (txhash: 829f421e381e4b84e4f1ac4ac121767eeb41d4280c555dc4cb1ae952307ee78f)

Follow Profile

Prefix: 0x8d06
Payload Parameters:
  • Address (35 bytes) - Legacy Address (Coming soon: RIPEMD160 of pubkey)

Important Note: Use the Legacy address, not Cash Address format (done for flexibility reasons)

Set Profile Header

Prefix: 0x8d08
Payload Parameters:
  • Header URL (77 bytes) - Image file URL or IPFS hash ('QmXaWRFhu6G6yCcy7dftsiehY8obJmX17hRtZr7BHyCbZC' in place of a URL)

Example:
OP_RETURN (HEX) 6a02 8d08 4568747470733a2f2f697066732e706963732f697066732f516d5861575246687536473679436379376466747369656859386f624a6d5831376852745a723742487943625a43
Sets profile header to URL https://ipfs.pics/ipfs/QmXaWRFhu6G6yCcy7dftsiehY8obJmX17hRtZr7BHyCbZC (In this case we used a proper URL, however simply 'QmXaWRFhu6G6yCcy7dftsiehY8obJmX17hRtZr7BHyCbZC' is also acceptable to allow clients to choose their IPFS gateway of choice

Create Media Post

Prefix: 0x8d09
Payload Parameters:
  • Media Type Identifier (1 byte) - Media Types (RESERVED = 0x00, IMAGE = 0x01,+ others in future)
  • Media URL/IPFS Hash (Variable length) - Media file URL. IPFS length is based on hash function. Standard hash example of 'QmXaWRFhu6G6yCcy7dftsiehY8obJmX17hRtZr7BHyCbZC' is 34 bytes when hex encoded. (Alternatively you may use a URL)
  • Text (caption for media) (73 - Media URL/IPFS Hash) - Optional parameter.

Example 1 (with text caption):
OP_RETURN (HEX) 6a02 8d09 01011668747470733a2f2f6269742e6c792f32465955726561304e65772023666561747572653a2023696d61676520706f737473206f6e20426c6f636b50726573732e20456e6a6f7921
Sets media to URL https://bit.ly/2FYUrea with caption 'New #feature: #image posts on BlockPress. Enjoy!'.
Blockchair Block Explorer

Set Profile Avatar

Prefix: 0x8d10
Payload Parameters:
  • Avatar URL (77 bytes) - Image file URL or IPFS hash (Ex: 'QmXaWRFhu6G6yCcy7dftsiehY8obJmX17hRtZr7BHyCbZC')

Example:
OP_RETURN (HEX) 6a02 8d10 1f68747470733a2f2f692e696d6775722e636f6d2f4a306b5239506d2e6a7067
Sets profile avatar to URL https://i.imgur.com/J0kR9Pm.jpg
Blockchair Block Explorer

Create Post in Community

Prefix: 0x8d11
Payload Parameters:
  • Community name (Variable length bytes) - Longer the name, the shorter the message can be
  • Message (74 - Community name length)

Example:
OP_RETURN (HEX) 6a02 8d11 0274762d4c6f73743a2054686520436f6d706c65746520436f6c6c656374696f6e202d20536561736f6e732031202d2036
Post message 'Lost: The Complete Collection - Seasons 1 - 6' to community 'tv'
Blockchair Block Explorer
